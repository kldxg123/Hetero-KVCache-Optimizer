# Hetero-KVCache-Optimizer: Artifact Evaluation Package

This repository contains the artifact for **Hetero-KVCache-Optimizer**, a tiered storage system for long-context MLLM inference on memory-constrained edge devices.

## Quick Start (Single Entrypoint)

The artifact provides a single entrypoint that runs the full pipeline:

```bash
bash run_all_experiments.sh
```

### Option A: Docker (Recommended)

```bash
# Build the artifact image
docker build -t hetero-kv-ae .

# Run the full experiment pipeline
# Requires an NVIDIA GPU with CUDA 12.1+ and nvidia-docker
docker run --gpus all -it hetero-kv-ae
```

### Option B: Native Environment

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Run the full pipeline
bash run_all_experiments.sh
```

**System Requirements:**
- NVIDIA GPU with at least 16 GB VRAM
- CUDA 12.1+
- Python 3.10+
- ~50 GB disk space (for model downloads and experiment outputs)

## Experiment Pipeline

Running `bash run_all_experiments.sh` executes the following steps:

1. **Unit Tests & Sanity Checks**
   - Validate 4-bit quantization round-trip error (< 0.5%)
   - Validate in-place rolling window correctness

2. **Micro-benchmarks**
   - Memory scalability sweep (4K -> 128K tokens)
   - PCIe bandwidth throttling simulation (16 GB/s, 8 GB/s)

3. **Simulated Baseline Comparisons**
   - Hetero-KV vs. Native HF
   - Hetero-KV vs. StreamingLLM
   - Hetero-KV vs. vLLM (simulated swap space)

4. **Real SOTA Baselines**
   - Hetero-KV vs. vLLM PagedAttention (strict 16GB limit)
   - Hetero-KV vs. SGLang RadixAttention (strict 16GB limit)
   - Hetero-KV vs. StreamingLLM (64 Sink + 4096 Local)
   - Hetero-KV vs. Native HF (4-bit NF4)

5. **Multi-Model Validation**
   - Llama-3.1-8B-Instruct (text LLM)
   - InternVL2-8B (multimodal LLM)

6. **Bandwidth Sensitivity**
   - Edge-hardware latency model under synthetic PCIe constraints

7. **Paper Compilation (optional)**
   - Compile LaTeX source if `pdflatex` is available

All results are written to `experiments/` and `real_sota_benchmarks/results/`.

| Claim from Paper | Command / Script | Expected Output |

| **Industrial baseline comparison** | `bash real_sota_benchmarks/run_real_benchmarks.sh` | JSON reports with TTFT, TPOT, throughput, and peak memory for vLLM, SGLang, StreamingLLM, and Native HF |

## PCIe Bandwidth Simulation

To reproduce the edge-hardware latency model on a server GPU, use the bandwidth limiter:

```bash
python tests/multimodel_benchmark.py \
  --models llama \
  --token_targets 4096 8192 12000 \
  --bandwidth_gbps 16.0
```

This injects sleep-proportional delays into all HBM<->DRAM transfers, simulating a consumer GPU on PCIe Gen4 x8 (~16 GB/s).
